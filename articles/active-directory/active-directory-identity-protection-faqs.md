# <a name="azure-active-directory-identity-protection-faq"></a>Preguntas frecuentes sobre Azure Active Directory Identity Protection


## <a name="why-do-some-risk-events-have-closed-system-status"></a>¿Por qué algunos eventos de riesgo tienen el estado "Cerrada (sistema)"?

**R:** Son eventos que Azure Active Directory Identity Protection detectó y más adelante cerró porque dejaron de considerarse arriesgados. Estos eventos no cuentan para el nivel de riesgo del usuario. 

---

## <a name="do-i-need-to-be-a-global-admin-to-use-identity-protection-in-the-azure-portal"></a>¿Es necesario ser administrador global para usar Identity Protection en Azure Portal?
**R:** **No**. Puede ser lector de seguridad, administrador de seguridad o administrador global para usar Identity Protection.

---

## <a name="how-do-i-get-identity-protection"></a>¿Cómo se consigue Identity Protection?
**R:** Vea [Introducción a Azure Active Directory Premium](active-directory-get-started-premium.md) para obtener una respuesta a esta pregunta.

---

## <a name="what-happens-when-your-azure-active-directory-premium-2-trial-expires"></a>¿Qué ocurre cuando expira el período de prueba de Azure Active Directory Premium 2?

**R:** si la edición de evaluación de Azure Active Directory Premium 2 ha expirado en la edición de Azure Active Directory Free, Basic o Premium 1, y tenía habilitada la protección de identidad, también tendrá acceso a él, aunque no cumple las licencias.
